### **B) HEALTHCHECK** ✅
**Problema**: Deploy sem snapshots resultava em endpoint vazio  
**Solução**: Script que verifica presença e validade de team-window-5 data

**Arquivo novo**:
- `tools/healthcheck-match-stats.mjs` (150 linhas)

**Funcionalidade**:
- Lê `data/v1/calendar_7d.json`
- Procura por `team-window-5/*/` snapshots
- Valida JSON com campo `windows`
- Exit code 0 (PASS) ou 1 (FAIL)

**Log**:
```
✅ Team-window-5 snapshots found and valid
✅ HEALTHCHECK PASSED
```

---

### **C) WORKFLOW OBRIGATÓRIO** ✅
**Problema**: Nada garante seed + smoke + healthcheck antes de deploy  
**Solução**: Workflow GitHub Actions com 3 novas etapas bloqueantes

**Arquivo alterado**:
- `.github/workflows/radartips_update_data_api_football.yml`

**Novas etapas**:
```yaml
- name: Seed team-window-5 snapshots
  run: node tools/seed-team-window-5.mjs

- name: Smoke test team-window-5
  run: node tools/smoke-test-team-window-5.mjs

- name: Healthcheck match-stats endpoint
  run: node tools/healthcheck-match-stats.mjs
```

**Comportamento**: Se qualquer uma falha → workflow inteiro falha → deploy bloqueado

---

### **D) UI ANTI-MENTIRA** ✅
**Problema**: Modal pode mostrar "0.00" como se fosse dato real  
**Solução**: Validação em runtime - nunca mostra 0, mostra "Sem dados ainda"

**Arquivo alterado**:
- `assets/js/match-radar-v2.js`: Função `renderStatsAccordion()` (linhas 424-434)

**Lógica**:
```javascript
if (!homeHasData && !awayHasData) {
  panel.innerHTML = 'Sem dados ainda — aguarde próximas partidas';
  return;
}
```

**Base disclosure**: Mostra `—` quando `games_used_total = 0`

## 🔒 GARANTIAS FINAIS (4-IN-1 DEFENSE)

### 1. **Impossível Cache Antigo**
```
HTML sempre referencia: app.[hash].js + match-radar-v2.[hash].css
Navegador invalida cache quando hash muda
Se HTML não tiver hash correto: process.exit(1)
```

### 2. **Impossível Publicar Vazio**
```
Workflow falha se:
  ✗ seed-team-window-5.mjs não completa
  ✗ smoke-test falha
  ✗ healthcheck retorna FAIL
→ Deploy bloqueado até resolver
```

### 3. **Impossível Endpoint Vazio**
```
Healthcheck antes de R2 upload verifica:
  ✓ data/v1/team-window-5/ existe
  ✓ Snapshots têm campo 'windows'
  ✓ JSON é válido
Se falha: exit 1 → deploy bloqueado
```

### 4. **Impossível UI Mentir**
```
Se games_used_total = 0:
  → Modal mostra "Sem dados ainda"
  → Base disclosure mostra "—"
  → NUNCA exibe "0.00" como número real
```
